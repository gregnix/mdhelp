# mdhelp 4 — Build Guide

Build standalone binaries for Linux and Windows.

## Prerequisites

- `tclsh` 8.6+ (Linux, or Windows with BAWT)
- **Library layout** (see `README.md`). `build.tcl` uses the same resolution
  as the runtime bootstrap and copies from there into the VFS:
  - **Pure-Tcl libraries** in `../libs/common/<repo>/` — `tclutils`,
    `tkutils`, `docir`, `mdstack`, `pdf4tcllib`, `pdf4tcl`. Platform- and
    version-independent.
  - **Compiled, per-platform binaries** in `../libs/<os>-tcl<ver>/` — e.g.
    `linux-tcl8.6/`, `linux-tcl9.0/`, `windows-tcl8.6/`, `windows-tcl9.0/`,
    each containing `Img*/`, `imgtools*/`, `tclpdfium/`, and (Windows)
    `sqlite3*/`, `tdbc*/`.
  - Per-library override: `TCLUTILS_HOME`, `TKUTILS_HOME`, `DOCIR_HOME`,
    `MDSTACK_HOME`, `PDF4TCLLIB_HOME`; platform layer override: `MDHELP_LIBS`.
    Fallbacks: sibling dev checkout, then `~/lib/tcltk/<repo>/`.
- Build runtimes in `../runtimes/` (one directory above the project) —
  **fetched manually once**, shared between all projects. See the setup
  instructions below.

## Quick start

```bash
# One-time: populate runtimes/ (see runtimes/README.md)

# Build
tclsh build.tcl             # Linux + Windows
tclsh build.tcl linux       # Linux only
tclsh build.tcl windows     # Windows only
tclsh build.tcl clean       # remove artefacts
```

Output in `dist/`:

```
dist/
  mdhelp-linux-x86_64          # Linux binary  (~5-8 MB)
  mdhelp-windows-x86_64.exe    # Windows binary (~5-8 MB)
```

## How it works

A **starpack** is a single executable file:

```
[Tclkit runtime] + [VFS with app code] = [standalone binary]
```

`build.tcl` builds the VFS, then sdx.kit + Tclkit wrap everything into
one file. The Windows binary is **cross-compiled** on Linux.

One VFS serves both platforms (it ships all `vendors/imaging/<os>-tcl<ver>/`
layers); at startup `main.tcl` selects the layer that matches the running
interpreter via `$tcl_platform(platform)` and `[info tclversion]`.

## VFS contents (generated by build.tcl)

```
mdhelp.vfs/
  main.tcl                   # entry point (tm::path, auto_path, imaging selector)
  mdhelp_app.tcl             # app (from app/mdhelp.tcl, bootstrap lines stripped)
  mdhelp_editor.tcl          # every other app/*.tcl is copied verbatim (glob):
  mdhelp_nav.tcl             #   mdhelp_ui, mdhelp_search_ui, mdhelp_settings,
  mdhelp_ui.tcl              #   mdhelp_imagetool, deepl_helper, sidebyside, ...
  ...                        #   (new app files are picked up automatically)
  applib/                    # mdhelp-owned modules (from lib/tm/)
  apptm/                     # docir + mdstack + pdf4tcllib
                             #   (from libs/common/<repo>/, resolved like at runtime)
  vendors/pkg/pdf4tcl0.9.4.x/  # 3rd-party pdf4tcl (from libs/common/pdf4tcl/)
  vendors/imaging/             # per-platform compiled packages, one dir per
    linux-tcl8.6/              #   OS x Tcl version. Contains Img (image formats),
    linux-tcl9.0/              #   imgtools (smooth scaling), tclpdfium, and on
    windows-tcl8.6/            #   Windows sqlite3 + tdbc. main.tcl puts only the
    windows-tcl9.0/            #   matching layer on auto_path / tm::path.
  docs/                      # embedded documentation
```

`main.tcl` (generated):

```tcl
package require Tk
set appDir [file dirname [file normalize [info script]]]
tcl::tm::path add [file join $appDir applib]
tcl::tm::path add [file join $appDir apptm]
foreach pkgDir [glob -nocomplain -directory [file join $appDir vendors pkg] *] {
    if {[file isdirectory $pkgDir]} { lappend ::auto_path $pkgDir }
}
# Platform + version specific binaries (Img, imgtools, tclpdfium, sqlite3, ...).
set osTag [expr {$tcl_platform(platform) eq "windows" ? "windows" : "linux"}]
set imgDir [file join $appDir vendors imaging $osTag-tcl[info tclversion]]
if {[file isdirectory $imgDir]} {
    lappend ::auto_path $imgDir
    catch {tcl::tm::path add $imgDir}
}
source [file join $appDir mdhelp_app.tcl]
```

## Setting up runtimes (one-time)

`build.tcl` expects `../runtimes/` (one directory above the project).
This directory is shared between all projects.

```bash
mkdir -p ../runtimes
cd ../runtimes

# sdx.kit
wget https://chiselapp.com/user/aspect/repository/sdx/uv/sdx.kit

# Download and extract the BAWT Tclkits
# from https://www.tcl3d.org/bawt/apps.html
wget https://www.tcl3d.org/bawt/files/Apps/Tclkits/tclkits-8.6.17.7z
7z e tclkits-8.6.17.7z "*Linux*tk" "*win*tk*"
rm tclkits-8.6.17.7z
chmod +x tclkit-Linux64-tk
```

Result:
```
../runtimes/
  sdx.kit
  tclkit-Linux64-tk
  tclkit-win64-tk.exe
```

The Tcl version of the runtimes decides which `vendors/imaging/*-tcl<ver>`
layer is used at run time. With the 8.6 Tclkits above, `linux-tcl8.6` /
`windows-tcl8.6` are selected; switching to 9.0 Tclkits selects the
`*-tcl9.0` layers automatically (they are shipped in the same VFS).

## Setting up the library layout (one-time)

```
../libs/
  common/                    # pure Tcl, shared by all platforms
    tclutils/  tkutils/  docir/  mdstack/  pdf4tcllib/  pdf4tcl/
  linux-tcl8.6/   linux-tcl9.0/     # compiled: Img*, imgtools*, tclpdfium
  windows-tcl8.6/ windows-tcl9.0/   # compiled: Img*, imgtools*, tclpdfium,
                                    #           sqlite3*, tdbc*
```

The imaging bundle (`Img` + `imgtools`, per OS x Tcl version) is what makes
the in-app **Image Tool** (crop/resize) work in the built binary: `Img`
loads/saves formats, `imgtools` does smooth scaling.

## Notes

- **Image support**: PNG, JPEG, TIFF via `Img`; smooth scaling via `imgtools`;
  both are bundled per platform under `vendors/imaging/`. Interlaced PNGs are
  supported when `pdf4tcl` >= 0.9.4.28 is used (Adam7 de-interlacing).
- **aspell / hunspell** cannot be bundled (C binary plus dictionaries).
  These must be installed on the target system. Spell checking is
  automatically disabled if no checker is available.
- **Tcl 9**: `build.tcl` patches `package require Tk 8.6` to `8.6-`
  automatically.
- **Test the Windows binary**: `wine dist/mdhelp-windows-x86_64.exe`
- **Run directly without building** (requires Tcl/Tk on the system):
  `wish app/mdhelp.tcl` on Linux, or `wish app\mdhelp.tcl` on Windows
  with BAWT. In this mode the runtime bootstrap resolves libraries from
  `<NAME>_HOME`, a sibling checkout, `../libs/common/<repo>/`, or
  `~/lib/tcltk/<repo>/`, and the platform binaries from
  `../libs/<os>-tcl<ver>/` (override with `MDHELP_LIBS`).
