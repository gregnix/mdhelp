# mdhelp 4 — Build Guide

Build standalone binaries for Linux and Windows.

## Prerequisites

- `tclsh` 8.6+ (Linux, or Windows with BAWT)
- **Module setup as described in `README.md`** — either `~/lib/tcltk/<repo>/`
  or a sibling layout. `build.tcl` uses the same paths as the runtime
  bootstrap and copies from there into the VFS.
- Build runtimes in `../runtimes/` (one directory above the project) —
  **fetched manually once**, shared between all projects. See the setup
  instructions below.

## Quick start

```bash
# One-time: populate runtimes/ (see runtimes/README.md)

# Build
tclsh build.tcl             # Linux + Windows
tclsh build.tcl linux       # Linux only
tclsh build.tcl windows     # Windows only
tclsh build.tcl clean       # remove artefacts
```

Output in `dist/`:

```
dist/
  mdhelp-linux-x86_64          # Linux binary  (~5-8 MB)
  mdhelp-windows-x86_64.exe    # Windows binary (~5-8 MB)
```

## How it works

A **starpack** is a single executable file:

```
[Tclkit runtime] + [VFS with app code] = [standalone binary]
```

`build.tcl` builds the VFS, then sdx.kit + Tclkit wrap everything into
one file. The Windows binary is **cross-compiled** on Linux.

## VFS contents (generated by build.tcl)

```
mdhelp.vfs/
  main.tcl                   # entry point
  mdhelp_app.tcl             # app (from app/mdhelp.tcl, bootstrap lines stripped)
  mdhelp_editor.tcl          # (from app/)
  mdhelp_nav.tcl
  mdhelp_ui.tcl
  mdhelp_search_ui.tcl
  mdhelp_settings.tcl
  applib/                    # mdhelp-owned modules (from lib/tm/)
  apptm/                     # docir + mdstack + pdf4tcllib (from user install
                             # ~/lib/tcltk/<repo>/, located via the mdhelp
                             # bootstrap)
  vendors/pkg/pdf4tcl0.9.4.x/  # 3rd-party pdf4tcl (from ~/lib/tcltk/pdf4tcl/)
  docs/                      # embedded documentation
```

## Setting up runtimes (one-time)

`build.tcl` expects `../runtimes/` (one directory above the project).
This directory is shared between all projects.

```bash
mkdir -p ../runtimes
cd ../runtimes

# sdx.kit
wget https://chiselapp.com/user/aspect/repository/sdx/uv/sdx.kit

# Download and extract the BAWT Tclkits
# from https://www.tcl3d.org/bawt/apps.html
wget https://www.tcl3d.org/bawt/files/Apps/Tclkits/tclkits-8.6.17.7z
7z e tclkits-8.6.17.7z "*Linux*tk" "*win*tk*"
rm tclkits-8.6.17.7z
chmod +x tclkit-Linux64-tk
```

Result:
```
../runtimes/
  sdx.kit
  tclkit-Linux64-tk
  tclkit-win64-tk.exe
```

## Notes

- **aspell / hunspell** cannot be bundled (C binary plus dictionaries).
  These must be installed on the target system. Spell checking is
  automatically disabled if no checker is available.
- **Tcl 9**: `build.tcl` patches `package require Tk 8.6` to `8.6-`
  automatically.
- **Test the Windows binary**: `wine dist/mdhelp-windows-x86_64.exe`
- **Run directly without building** (requires Tcl/Tk on the system):
  `wish app/mdhelp.tcl` on Linux, or `wish app\mdhelp.tcl` on Windows
  with BAWT.
